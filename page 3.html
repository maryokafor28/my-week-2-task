SEMANTICS HTML AND NORMAL HTML

Semantics html are tags that define the meaning of the content they contain. for example, tags like;

  • < header >
  • < article >
  • < footer >

They indicate the role of the content they contain

NORMAL HTML;

Tells nothing about the content they contain. examples are;
  • < div >
  • < span >

The Semantics html helps the search engines and other user device to determine the
importance and context of web pages . The pages made with semantic element are much easier to read.

Semantics html has greater accessibility.
It offers great user experience.

Normal html do not have specific meaning. example; < div > tag that can be used to group elements together.

Normal Html serve only as a content holder but give no indication as to what type of content they contain or what role
the content plays on the page.

DISPARITY BETWEEN CLOSING TAGS AND SELF CLOSING TAGS

  • Closing tags necessitate the use of a closing tags. example; < > tag always need < /p >
  • Self closing tags do not necessitate a closing tag. Example;< inputs>
  • closing tags always proceed the element with a /
  • Self closing tags are referred to as empty elements. It does not require a forward slash. eg hr, br img.

DIFFERENCE BETWEEN AN INLINE ELEMENT AND A BLOCK ELEMENT IN HTML

  • A block element always start on a new line and the browser automatically add some space (a margin) before and after the element.
  • An inline element does not start on a new line. It takes up as much width as necessary.
